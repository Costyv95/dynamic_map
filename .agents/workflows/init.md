# /init

### Steps
1. Read `docs/map.md` and run `ls -R docs/` to understand the project structure.
2. Read the **essential design docs** (in this order):
   - `docs/original_idea.md` (brand identity, market context, economic constraints)
   - `docs/technical_design.md` (full system architecture, data model, deployment)
   - `docs/history.md` — read the **last ~200 lines** for recent ADRs only (if it exists)
3. Read the **5 most recent** files from `docs/handoffs/` (sorted by filename date).
4. Check **Knowledge Items** for cached context on key systems before doing further research.
5. Confirm: "I am synced with the project history. Should I focus on any specific area?" Flag any unfinished tasks from the latest handoff.
