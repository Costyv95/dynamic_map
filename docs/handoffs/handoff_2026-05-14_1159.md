# Session Handoff: Map Orientation & True Axis-Mirroring
**Date:** 2026-05-14 11:59

### Current Objective
The primary goal of this session was to fix critical desyncs in the Dynamic Map rotation and axis-mirroring systems. We needed to ensure that flips (horizontal/vertical) applied correctly relative to the user's *screen*, not the map's internal geometry, and that the "Auto" rotation mode strictly inherited the flip settings of its chosen target orientation without allowing manual override.

### Changes Made
*   **`CanvasEngine.js`**: 
    *   Moved the calculation of the map's true active orientation into `calculateAutoCrop` and exposed it via `getActiveMode()`. This ensures the UI and the rendering engine never disagree on what orientation is currently active.
    *   Added geometric inversion logic: whenever the map rotates 90 degrees (`isRotated`), we dynamically swap the application of the `X` and `Y` scale flips. This guarantees that "Horizontal Flip" always mirrors left-to-right on the screen, regardless of whether the map's source image is portrait or landscape.
*   **`custom-svg-map.js`**:
    *   Synchronized the Lovelace card renderer to use the exact same `scaleX` / `scaleY` inversion logic as `CanvasEngine`, ensuring the dashboard perfectly matches the editor.
*   **`editor.js`**:
    *   Updated the UI bindings to fetch the flip state from `engine.getActiveMode()` rather than calculating percentages on its own, eliminating the desync bug where clicks modified the wrong configuration array.
    *   Locked the flip buttons to be strictly read-only (but visually active) while in "Auto" mode.
*   **`editor.html`**:
    *   Upgraded the flip buttons to use standard vector `arrow-left-right` and `arrow-up-down` icons, significantly improving intuitive UX.

### Blocked/Pending Tasks
*   **Phase 2 Architecture Refactor**: We need to begin decoupling the bloated logic within the Lovelace component.
    *   Extract viewport panning/zooming into a dedicated `CameraManager`.
    *   Isolate the `VacuumTracker` to prevent frame-rate drops during map interactions.
    *   Modularize the SVG generation into a standalone `MapRenderer` class.

### Architectural Shift
*   **Decoupled Flip Configuration:** We officially transitioned away from global `horizontal_flip` and `vertical_flip` booleans to a nested object structure (`flips: { horizontal: { h, v }, vertical: { h, v } }`). This structurally prevents "orientation bleed" and maintains user preferences flawlessly between viewports.
