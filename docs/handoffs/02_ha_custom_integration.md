# Handoff: Home Assistant Custom Integration Migration

## Objective
The primary objective of this phase was to mature the project from a local Python script running alongside a standalone HTML file into a fully native, HACS-compliant Home Assistant Custom Integration. This shift provides secure persistence of configuration data and embeds the editor directly into the Home Assistant UI.

## Key Accomplishments

### 1. Integration Architecture Implementation
- **Project Restructuring:** Transitioned the codebase into a standard Home Assistant structure (`custom_components/dynamic_map/`).
- **Metadata Generation:** Created the required `manifest.json` outlining the integration's dependencies (`http`, `frontend`) and metadata.
- **Python Backend (`__init__.py`):** Authored the core integration logic. The backend performs three critical tasks upon initialization:
    1. Registers a static path (`/dynamic_map_ui`) mapping to the `frontend/` directory, allowing Home Assistant's web server to host the static assets (HTML, JS, SVGs).
    2. Registers an iframe-based Custom Panel, injecting the "Map Editor" directly into the user's main Home Assistant sidebar.
    3. Registers custom HTTP Views (`DynamicMapSaveView`, `DynamicMapStateView`) exposing the `POST /api/dynamic_map/save` and `GET /api/dynamic_map/state` endpoints.

### 2. Secure API Layer
- **API Endpoints:** The `DynamicMapSaveView` securely intercepts POST requests from the frontend, while `DynamicMapStateView` provides secure read access to entity states. Both include built-in Home Assistant authentication requirements (`requires_auth = True`). The save endpoint includes strict directory traversal protections.
- **Direct File I/O:** When triggered, the backend decodes the incoming JSON payload and writes `rooms.json` and `shortcuts.json` directly back into the `frontend/` directory on the host machine. This entirely replaces the need for the legacy local Python server.

### 3. Frontend Decoupling
- **`editor.html` Refactor:** Stripped out the legacy local export mechanisms (which relied on forcing a browser file download via `data:text/json`).
- **API Integration:** Rewrote the "Save" action to use standard asynchronous `fetch()` requests pointing to the new `/api/dynamic_map/save` endpoint. Because the editor is hosted within an iframe generated by Home Assistant, it implicitly inherits the user's active session, meaning authentication is seamlessly handled without requiring manual API tokens.

### 4. HACS Compatibility
- Built the `hacs.json` and `README.md` metadata files required by the Home Assistant Community Store, allowing the project to be deployed and updated as a Custom Frontend/Integration Repository.
