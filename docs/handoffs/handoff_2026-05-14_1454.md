# Handoff: Vacuum Orchestration & Map Refinement
**Date:** 2026-05-14 14:54

## Current Objective
Finalize the vacuum orchestration and map logic by resolving runtime errors in the `ROOM_SELECTOR` action, properly supporting Roborock's hidden integer Segment IDs for `app_segment_clean` service calls, and ensuring the SVG map acts cleanly as both a tracker and a command dispatcher.

## Changes Made
- **custom_components/dynamic_map/__init__.py:**
  - Added a new Python backend API view `DynamicMapRoborockRoomsView` (`/api/dynamic_map/roborock_rooms`) that acts as a secure proxy to bypass frontend restrictions and calls `roborock.get_maps` with `return_response=True` to fetch the vacuum's segment dictionary.
- **custom_components/dynamic_map/frontend/ApiManager.js:**
  - Overhauled `fetchVacuumRooms` to aggressively scrape HA for segment IDs.
  - Implemented 4 tiers of scraping: 1) `vacuum` attributes, 2) `camera` map attributes, 3) the new python proxy for `roborock.get_maps`, 4) `sensor.current_room` state tracking names.
- **custom_components/dynamic_map/frontend/editorUI.js:**
  - Added "Seg ID" input fields to the Room Mapping UI so users can explicitly bind integer segment IDs to strings for dual tracking-and-command capabilities.
  - Added auto-population logic that fills the "Seg ID" field automatically when `ApiManager` successfully finds the data.
- **custom_components/dynamic_map/frontend/custom-svg-map.js:**
  - Fixed a UI crash (`this.closeOverlay is not a function`) inside the `ROOM_SELECTOR` action click listener.
  - Updated payload parsing and `ROOM_SELECTOR` segment generation to properly map string room names to integer Segment IDs before dispatching `vacuum.send_command`.
- **custom_components/dynamic_map/frontend/shortcuts/GenericShortcut.js:**
  - Fixed an issue where the fallback emoji/text icon wouldn't reappear if an external SVG/PNG image threw a 404 error, by implementing a proper `<image>` `error` event listener.

## Blocked/Pending Tasks

## Architectural Shift
- The map logic now permanently decouples State Tracking strings (e.g. `"Kitchen"`) from Service Command IDs (e.g. `16`). 
- A secure Python-to-Frontend Service Proxy pattern was established in `__init__.py` to allow the Lovelace card to extract rich data from HA services that require `return_response=True`.
