# Session Handoff: 2026-05-11 08:40

## Current Objective
The primary goal of this session was to stabilize the Dynamic Map Custom Card UI on the Home Assistant dashboard. This involved fixing layout scaling issues, matching the rendering behavior exactly to the standalone Map Editor, and addressing cache-invalidation challenges.

## Changes Made
- **Layout Hardening (`custom-svg-map.js`)**: Replaced restrictive Flexbox centering with absolute positioning on the `renderRoot` to firmly anchor the map. Removed `aspect-ratio` hacks, allowing the SVG `viewBox` to scale naturally to 100% width and height.
- **Responsive Geometry**: Implemented a `ResizeObserver` to dynamically trigger `calculateAutoCrop()` whenever the Home Assistant panel resizes, fixing the "gravity well" and infinite-scroll layout bugs.
- **Visual Seamlessness**: Set the `renderRoot` background to solid white (`#ffffff`) to perfectly match the floorplan image background, preventing the gray dashboard background from bleeding through during map panning.
- **Shortcut & Vacuum Standardization**: Rewrote the shortcut rendering logic in `custom-svg-map.js` to use fixed pixel scales (`rx = 12 * scaleX`) to perfectly match `editor.html`. Rewrote the vacuum icon using the three-tier circle logic bounded correctly within its rect.
- **Rotation Synchronization**: 
  - Added a shared top-left UI container (Flexbox) and ported the Rotation Mode (`AUTO`, `Horizontal`, `Vertical`) toggle button directly onto the dashboard map.
  - Removed counter-rotation logic so all overlay objects (room labels, shortcuts, and the vacuum icon) rotate naturally alongside the base map.
- **Cache Busting / Versioning**: Bumped the integration version to `v2.2.0` in `manifest.json`, the iframe URL in `__init__.py`, and the visual SVG badge.

## Blocked/Pending Tasks
1. **Frontend Directory Cleanup**: The `custom_components/dynamic_map/frontend/` directory still contains legacy Python scripts (e.g., `dxf_processor.py`) that must be moved to a `scripts/` directory.
2. **Backend Hardening**: JSON schema validation needs to be implemented in `__init__.py` before saving configurations to the filesystem.
3. **Cache Invalidation UX**: Despite codebase version bumps, Home Assistant heavily caches the frontend resource. Users must manually update the query parameter in their Lovelace resources (`?v=2.2`). A fully native `custom-svg-map-editor.js` would alleviate this iframe/cache complexity long-term.

## Architectural Shift
- **Geometry Source of Truth**: The map's layout engine now relies entirely on `getBoundingClientRect()` via `ResizeObserver` to determine its aspect ratio constraints, rather than fighting the parent Home Assistant container with static CSS variables.
