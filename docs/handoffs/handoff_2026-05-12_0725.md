# Handoff: Dynamic Map Recomputation & Microservice Architecture
Date: 2026-05-12

## Current Objective
The primary goal of this session was to integrate the DXF-to-JSON automated map generation pipeline directly into the Home Assistant frontend Map Editor, allowing users to regenerate floorplans on the fly without running python scripts on their personal computers.

## Changes Made
1. **Recompute Map API**: Added `/api/dynamic_map/files` and `/api/dynamic_map/recompute` to `__init__.py`.
2. **AI Stripped**: Removed the heavy `transformers` and CLIP models from `dxf_processor.py` to drastically reduce dependency weight. Rooms are now automatically named "Room 1", "Room 2", etc.
3. **Editor UI Upgrade**: Built a new "Advanced: Recompute Floor" panel in `editor.html` to allow selecting uploaded `.dxf` and `.svg` files to trigger a backend regeneration.
4. **Alpine Dependency Fix**: Removed `opencv-python-headless` and `cairosvg` from `manifest.json`. These C++ libraries cannot be compiled in the Home Assistant Alpine Docker environment and cause integration loading failures. They are now dynamically imported in `dxf_processor.py` with clear fallback errors.
5. **Project Cleanup**: Deleted all old `test_*.py` files and obsolete debug scripts (`crop.py`, `dxf_peek.py`, etc.) from the `frontend/` directory.
6. **Server Migration Documentation**: Created `docs/server.md` detailing the migration path from a Raspberry Pi to an HP EliteDesk 800 G5 Mini PC using Proxmox VE and a separate Docker LXC container for the OpenCV sidecar microservice.

## Architectural Shift
We encountered a hard architectural limitation: Home Assistant OS is a locked-down Alpine environment that does not support heavy C++ compilation for packages like OpenCV. 

As a result, we pivoted from running the OpenCV backend natively within Home Assistant, to a **Microservice Architecture** detailed in `docs/server.md`. The map recomputation will eventually be offloaded to a secondary Docker sidecar container running on Proxmox, which can instantly install Debian pre-compiled x86-64 wheels.
